# Product

## Register

brand

## Users

Ewan himself (primary), visitors reading posts or checking status (secondary), AT Protocol developers referencing the implementation (tertiary). Visitors arrive from Bluesky links, search, or direct navigation — they're browsing, not working.

## Product Purpose

Personal site, blog, and digital garden powered by AT Protocol. Displays profile, mood, music, posts, supporters, and publications in a single masonry dashboard. Serves as both a living homepage and a reference implementation for AT Protocol-powered personal publishing.

## Brand Personality

Technical, quiet, present. Three words: _precise, forested, lived-in_.

The site should feel like a well-maintained terminal in a cabin — mechanical where it needs to be, organic where it breathes. Not a portfolio pitch, not a startup landing. A corner of the internet that someone actually lives in.

## Anti-references

- **Startup landing pages.** Hero sections with gradient CTAs, testimonial carousels, pricing tables. This is not a product.
- **Generic portfolio templates.** Centered photo + name + social links in a card. That shape is everywhere.
- **Over-animated agency sites.** Scroll-jacking, parallax overload, loading screens. Motion serves clarity, not spectacle.
- **Corporate minimalism.** White space for the sake of white space. Inter at 16px with nothing to say.

## Design Principles

1. **Lived-in, not launched.** The site should feel like it's been here a while, not like it just shipped. Content accumulates. Status updates change. The page breathes.
2. **Structure is mechanical, content is organic.** Cards, grids, and navigation are precise and technical. The things inside them — moods, music, writing — settle naturally.
3. **Show the wire.** AT Protocol is part of the identity, not hidden infrastructure. The firehose is live. The data comes from a PDS. This is a feature, not an implementation detail.
4. **Restraint with intent.** Empty space exists because something was deliberately removed, not because nothing was added. Every element earns its place.
5. **Terminal meets forest.** The aesthetic is a dark green terminal in a forest clearing. Sharp edges, monospace accents, green glow — but also natural deceleration, quiet transitions, calm presence.

## Accessibility & Inclusion

- WCAG 2.1 AA minimum
- `prefers-reduced-motion` fully respected — all animations collapse to near-zero duration
- `prefers-color-scheme` respected for initial dark/light mode
- Focus-visible outlines on all interactive elements
- Skip-to-content link
- Semantic HTML throughout (landmarks, headings, ARIA labels)
- Colour themes use OKLCH for perceptual uniformity across light/dark
