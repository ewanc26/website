## Design Context

### Users

A technical, personal audience looking to understand the creator's projects, thoughts, and technical exploration.

### Brand Personality

1.  **Technical**: Precision, functional integrity, systematic.
2.  **Personal**: Curated, singular perspective, Anglo-Scottish roots.
3.  **Pagan**: Nature-aligned, seasonal, traditional yet evolving.
4.  **Playful**: Hidden easter eggs, joy in the details (e.g., Wolf Mode).

### Aesthetic Direction

Minimalist, systematic, and typography-led with a "Modern Pagan" influence. The design is deeply tied to the "Wheel of the Year," creating a living interface that shifts with the seasons.

### Design System Definition

#### 1. Wheel of the Year (Seasonal Dynamic Engine)

- **Sabbat System**: The interface tracks 8 major Sabbats (Imbolc, Ostara, Beltane, Litha, Lughnasadh, Mabon, Samhain, Yule).
- **Dynamic Theming**: All colours are dynamically generated in OKLCH, interpolating between the previous and next Sabbat.
- **Branding**: Neutrals are tinted toward the current seasonal brand hue (chroma 0.005–0.012).
- **Visuals**: Sabbat icons wax and wane in the background of major sections.

#### 2. Colour (OKLCH & Tailwind v4)

- **Engine**: Dynamic CSS generation at the server level via `src/lib/server/theme.ts`.
- **Contrast**: High-legibility light and dark modes using `light-dark()`.
- **Palette**: No pure black/white. Solid colours only; gradients are restricted to seasonal background icons.
- **Framework**: Tailwind CSS v4 provides the utility-first layer over the dynamic tokens.

#### 3. OpenGraph (1200x630)

- **Generator**: Dynamic PNG generation via `@resvg/resvg-js` at `/api/og/generate`.
- **Branding**: "ewancroft.uk" is forced in the bottom-left corner with a 10% opacity Pentacle icon.
- **Fallbacks**: Type-based fallbacks for missing titles (e.g., `BLOG` -> "Blog", `DESIGN` -> "Design").
- **Aesthetic**: Force dark mode for social previews.

#### 4. Typography (Variable Fonts)

- **Primary**: **Inter Variable** for all UI and body text.
- **Mono**: **JetBrains Mono** for code, metadata, and technical badges.
- **Scale**: Modular scale (ratio 1.25), fluidly sized using `clamp()`.
- **Constraints**: Narrow prose columns (max 65–75ch) for readability.

#### 5. Spacing (4pt Scale)

- **Scale**: 4, 8, 12, 16, 24, 32, 48, 64, 96.
- **Layout**: Heavy reliance on `gap` and `margin-inline: auto` (shells).
- **Shells**: `shell-wide` (72rem), `shell-prose` (72rem), and `shell-narrow` (48rem).

#### 6. Localisation

- **Locale**: Strictly localised to `en_gb` across all HTML attributes, OpenGraph metadata, and user-facing date formatting.

### Design Principles

1.  **Seasonal Fluidity**: The interface is never static; it breathes with the cycle of the year.
2.  **Systematic Rigour**: Every token is derived from a core mathematical scale (4pt spacing, 1.25 type ratio).
3.  **Playful Seriousness**: Technical excellence paired with whimsical features like "Wolf Mode" (text-to-howl transformation).
4.  **Anti-Slop**: Absolute ban on AI-typical patterns (side-stripe borders, floating cards without intent).
5.  **Functional Symbolism**: Symbols (Pentacle, Triskele, Leaflet) are used as functional markers and end-of-post signals.
