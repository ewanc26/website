## Design Context

### Users

A technical, personal audience looking to understand the creator's projects, thoughts, and technical exploration.

### Brand Personality

1. **Technical**: Precision, functional integrity.
2. **Personal**: Curated, singular perspective.
3. **Exploratory**: Open to new technologies.

### Aesthetic Direction

Minimalist, systematic, and typography-led. Every element serves a function.

### Design System Definition

#### 1. Color (OKLCH)

- **Base Palette**: Derive all colors from brand-specific OKLCH hues.
- **Neutrals**: Tint neutrals toward the brand hue (chroma 0.005-0.01).
- **Theme**: Support light/dark modes using `light-dark()` CSS function.
- **Constraints**: No pure black/white. No AI-standard palettes. Solid colors only.

#### 2. Typography (Variable Inter)

- **Scale**: Modular scale, ratio 1.25. Fluid sizing using `clamp()`.
- **Hierarchy**: Clear weights and contrast. Line-height scales inversely with length.
- **Constraints**: Narrow columns (max 65-75ch). No all-caps for body.

#### 3. Spacing (4pt Scale)

- **Scale**: 4, 8, 12, 16, 24, 32, 48, 64, 96.
- **Tokens**: Semantic naming (e.g., `--space-sm`, `--space-md`).
- **Layout**: Use `gap` for all sibling spacing.

### Design Principles

1. **Systematic Color**: OKLCH usage, brand-hue tinting.
2. **Purposeful Spacing**: 4pt scale, semantic tokens.
3. **Typography as Hierarchy**: Strict ratio-based scale.
4. **Functional Interactivity**: Motion for state changes only.
5. **Anti-Slop**: Absolute ban on AI-typical patterns (side-stripe borders, gradient text).
